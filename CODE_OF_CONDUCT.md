# Code of Conduct

This repository, the Hooked Bears Twitter/X account, and any future community spaces we run are committed to being **a useful place to think about NFTs, tokens, and on-chain mechanics — without the toxicity, hype, or hostility that often comes with crypto.**

This Code of Conduct applies to all contributors, maintainers, and visitors.

## 🤝 Our standards

**Examples of behavior that helps the community:**

- Being patient with new users — everyone was confused by their first wallet
- Crediting people whose work or ideas you build on
- Disagreeing with the *idea*, not attacking the person
- Asking questions before assuming bad faith
- Sharing knowledge (especially about on-chain verification, contract reading, and security)
- Reporting scams, phishing, and impersonation accounts
- Accepting that the maintainer is a solo, pseudonymous developer with limited bandwidth

**Examples of behavior we won't tolerate:**

- Personal attacks, harassment, slurs, or doxxing attempts
- Sustained price-pumping, FUD campaigns, or coordinated raids
- Sharing or soliciting others' seed phrases, private keys, or sensitive personal info
- "Support DMs" — only scammers DM first claiming to be support
- Promoting unrelated tokens, NFT projects, or affiliate links in issues, PRs, or discussions
- Posting graphic, sexual, or violent imagery
- Trolling, deliberately bad-faith arguments, or spam
- Encouraging others to violate this Code of Conduct

## ⚖️ Scope

This Code applies whenever you're representing the project — in this repo, in reply threads on the official Twitter/X account, in any future community spaces, and at any real-world event tied to the project.

It also applies to private DMs *between contributors* when those DMs are about Hooked Bears matters.

## 🪧 Reporting

If you see or experience behavior that violates this Code:

- **In this repository** — open a [private vulnerability report](../../security/advisories/new) (yes, we use the same channel for both security and CoC issues; it's the most private path). Or DM [@hookedbears](https://x.com/hookedbears) on X.
- **On X / Twitter** — block the offender first, then DM us with screenshots so we know who to block on our end.
- **About the maintainer** — if your concern is about the project lead, document it publicly (with screenshots) on X and tag us. Solo-dev projects need outside scrutiny.

All reports are read. We will not retaliate against good-faith reporters even if we conclude no violation occurred.

## 🔨 Enforcement

We try to be proportionate:

| Behavior | Typical response |
|---|---|
| Heated disagreement that crosses into personal attack | Public note to cool down, edit/delete request |
| Off-topic shilling | Comment removed, contributor warned |
| Repeated or escalating violations | Temporary lock from issues/PRs (up to 30 days) |
| Doxxing, harassment, scam attempts, sharing CSAM or violent content | Permanent ban + report to GitHub Trust & Safety |
| Phishing or wallet-drain attempts | Permanent ban + public warning to the community |

Decisions are made by the project maintainer. We try to be transparent about why an action was taken when it's safe to do so.

## 🛟 Attribution & inspiration

This Code is influenced by the [Contributor Covenant](https://www.contributor-covenant.org/) and the [Geek Feminism Anti-Harassment Policy](https://geekfeminism.fandom.com/wiki/Conference_anti-harassment/Policy), adapted for a small, crypto-adjacent, single-maintainer project.

We didn't copy the Contributor Covenant verbatim because we wanted a CoC that:

1. Specifically names the bad behaviors that show up in crypto communities (scam DMs, shilling, FUD raids)
2. Acknowledges the asymmetric power dynamic of a single pseudonymous maintainer
3. Is short enough to actually be read

## 💬 In summary

Treat people like you're sitting across a table from them. Help newcomers. Disagree honestly. Don't drain anyone's wallet. Don't promote your shitcoin in our issues.

Thanks for being here. 🐻🪝
